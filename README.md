## 0. 프로젝트 개요

본 프로젝트는 공통 요구사항을 기반으로 한 칸반 형태의 태스크 관리 대시보드 구현 과제입니다.

CRUD, 기간 필터, 통계, 테마 및 UX 요소를 포함하며 팀 단위 협업을 통해 설계 및 구현을 진행하였습니다.

## 2. 수행 절차 및 방법 (Process & Strategy)

### 2-1. 진행 순서

1. 요구사항 전체 리뷰 및 필수 / 가산 요구사항 구분
2. 데이터 모델 및 LocalStorage 키 합의
3. 보드 구조 및 렌더링 전략(State-driven) 결정
4. 필터 / 정렬 / 검색 파이프라인 구현
5. UX / 테마 / 반응형 구현
6. 통합 및 최종 QA

### 2-2. 협업 규칙

- 브랜치 전략: `main` / `dev` / `feature/*`
- PR 단위: 기능 1개 기준
- PR 전 `rebase` 필수
- 스펙 변경은 `dev` merge 전까지만 허용

## 3. 프로젝트 구조 및 아키텍처

### 3-1. 디렉터리 구조

```
flowdash/
├─ README.md
├─ index.html
├─ css/
│  ├─ reset.css
│  ├─ base.css
│  ├─ components.css
│  ├─ theme.css
│  └─ responsive.css
└─ js/
   ├─ main.js
   ├─ config.js
   ├─ selectors.js
   ├─ state.js
   ├─ storage.js
   ├─ utils/
   │  └─ date.js
   └─ ui/
      ├─ board.js
      ├─ card.js
      ├─ header.js
      ├─ modal.js
      └─ stats.js
```

### 3-2. 데이터 흐름

- **User Action** (사용자 입력/클릭)
- → **updateState()** (상태 변경)
- → **saveToStorage()** (로컬 스토리지 저장)
- → **renderBoards()** (필터/정렬 적용 후 화면 갱신)
- → **renderStats()** (통계 업데이트)

## 6. 트러블슈팅 (Troubleshooting)

### 6-1. (github 최신화 이슈)

- **증상**: +새 할 일: 클릭 후, 모달창에서 저장하기 안됨, 검색&필터에서 우선순위 및 날짜, 정렬 칸반목록이 나오지 않음.
- **원인**: github - git pull 작업 원활화 되지 않아 github는 최신화 / Local에서는 이전 커밋단계 상태
- **해결**: git status 및 git rebase origin/feature flowdash로 터미널에 입력하여 최신화 됨
- **회고**: github는 초보자로서는 난이도가 있는 프로그램이면서 꼼꼼한 프로그램인 것 같아 더 알아보면서 부딪치는 존재인 것 같다.

### 6-2. (트러블보다는 배경이미지 선택)

- **증상**: body 배경이미지가 흰색이어서 랜덤 이미지 URL 사진으로 타임을 20초정도 코드펜에 시연하여 진행하였음.
- **원인**: 랜덤이미지 URL 사진들이 기능구현된 메뉴아이콘들보다 눈에 띄어서 의견조율
- **해결**: 은은한 색상 그라데이션으로 선택하여 기능구조에 구현우선시 함.
- **회고**: 다양한 의견으로 적절한 선택을 할 수 있어서 좋았다.

### 6-3. (파일의 모듈화)

- **증상**: Ai프로그램과 강사님의 도움을 받아 파일의 모듈화를 진행하였으나 몇가지 기능의 누락이 발생
- **원인**: 제대로 연결되지 않은 js, 클래스 명 불일치, 데이터 누락
- **해결**: 연결되지 않은 것들은 연결해주고, 클래스명들 일치하도록 수정하고, 누락된 데이터들은 전부 다시 찾아 추가했습니다
- **회고**: 모듈화 작업하는게 쉽지 않았고 데이터 누락시키지 않도록 하나하나 제대로 살펴봐야겠다고 느꼈으며 클래스 명도 변경된 부분 제대로 확인해야겠다고 생각함

### 6-4. (다크모드 적용 오류)

- **증상**: 다크모드를 적용했으나 제대로 적용되지 않는 부분들 발생
- **원인**: git협업중 내려받은 파일에서 수정된 부분 인지못함, 변수로 설정해야하는 부분에 값을 넣어줌
- **해결**: 값이 들어간 부분 변수로 변경, 서로 조원끼리 push했다고 말해주기
- **회고**: 소통을 잘해야겠다고 느꼈고 누락시키는 부분들 항상 조금씩 있는거 같아 꼼꼼히 살펴보도록 주의해야겠다고 생각함

### 6-5 (css 코드 일부 적용 오류)

- **증상**: 팀원들과 협업과정에서 코드와 겹치면서 일부 스타일이 적용되지 않는 문제 발생
- **원인**:상대 코드의 CSS가 우선 적용되는 충돌 현상 발생 CSS 우선순위 및 로드 순서 문제로 내가 작성한 스타일이 덮어씌워짐
- **해결**:멘토님의 피드백을 바탕으로 문제 원인을 파악하고 클래스명을 구분하여 스타일 충돌을 방지 필요 요소에 대해 CSS 우선순위 구조를 재정리하여 의도한 스타일이 정상 적용되도록 수정
- **희고**:실무와 유사한 이슈를 경험하며 디버깅 능력과 협업의 중요성을 배운 프로젝트였고 어려운 문제는 멘토님의 피드백을 적극적으로 활용해 해결하며 소통과 피드백을 통한 문제 해결 과정이 개발 완성도를 높인다는 점을 느꼈다

## 7. 자체 평가 및 회고 (Self Review)

### 7-1. 잘한 점

- 기한 안에 기능구현이 100% 잘 되게 설계된게 좋다
- 처음 접속시 닉네임 받는 모달을 만든건 잘한것 같다
- 팀 프로젝트에서 맡은 기능을 기한 내 완성하며 전체 흐름에 맞게 연동되도록 구현했다.

### 7-2. 아쉬운 점

- 해당 기능에 다른 코드를 더 넣을 수 있는지 고민할 시간이 부족한 것 같다.
- 다른 부분에도 닉네임 모달같이 추가 기능을 넣을 수 있었으면 좋았을텐데 시간이 모자라서 아쉬웠다.
- 기능 완성하느라 디자인 시간이 거의 없어서 디자인에 대한 고민을 거의 못한것이 아쉬웠다.
- 조장이었는데 내가 맡은 부분 하는거에 바빠서 자기 몫을 완료한 사람에게 추가적인 역할과 할일을 정확하게 나눠주지 못한거 같아서 아쉬웠다.
- 협업 시 코드 스타일과 규칙을 미리 맞추지 못해 CSS 충돌 문제가 발생한 점이 아쉬웠다.

### 7-3. 다음에 개선할 점

- 다른 조원의 역할에 대한 공부 부분이 필요하게 되었습니다
- 혹시나 다시 조장을 한다면 시간을 제대로 활용할 수 있도록 조원들 능력을 파악하고 적절한 역할을 부여해야겠다
- 다양한 추가 모달과 디자인 쪽을 개선해보고 싶다
- 기능 구현 전 간단한 구조 설계와 역할 분담을 통해 작업 효율을 높이고 싶다.
